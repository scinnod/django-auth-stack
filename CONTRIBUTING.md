# Contributing

This is an open-source project developed at [Jade University of Applied Sciences](https://www.jade-hs.de/).

Contributions are welcome! However, please note that this is a **university infrastructure project** with limited resources for support.

## Philosophy

This project is **free to use** under the Apache-2.0 license. We share it to help others implement similar authentication solutions. However:

- **Support resources are limited** - We cannot provide extensive individual support
- **Contributions are valued** - We welcome contributions from those who benefit from the project
- **Give back** - If you use this project, consider contributing improvements back to the community

## How to Contribute

### Reporting Issues

1. Check existing issues first to avoid duplicates
2. Provide clear reproduction steps
3. Include your environment details (Docker version, OS, etc.)
4. Be patient - responses may take time

### Submitting Changes

1. **Open an issue first** to discuss proposed changes
2. Fork the repository
3. Create a feature branch (`git checkout -b feature/your-feature`)
4. Make your changes with clear commit messages
5. Test your changes thoroughly
6. Submit a pull request with a clear description

### What We're Looking For

- Bug fixes with clear explanations
- Documentation improvements
- Security enhancements
- Compatibility updates (new Keycloak/Docker versions)
- Translations of documentation

### What We Cannot Accept

- Changes that break backward compatibility without discussion
- Features that significantly increase complexity
- Changes without adequate documentation

## Code of Conduct

- Be respectful and constructive in all interactions
- Focus on the technical merits of contributions
- Respect that maintainers have limited time
